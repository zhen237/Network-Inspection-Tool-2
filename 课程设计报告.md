## 摘  要

本课程设计针对eNSP（Enterprise Network Simulation Platform）网络设备管理需求，设计并实现了一个基于Web的智能化设备管理与自动巡检平台。系统采用Python Flask作为后端框架，整合Socket.IO实现浏览器与服务器的实时双向通信，通过Telnet协议与eNSP虚拟设备建立连接。核心功能包括：设备自动发现与扫描、多设备连接管理、一键巡检报告生成、配置比对分析。同时，系统集成MCP（Model Context Protocol）协议，支持与AI助手无缝对接，为网络运维提供智能化支持。

实际应用表明，该平台显著提升了网络设备管理效率，降低了手工巡检的时间成本，为网络工程实践教学提供了有力的技术支撑。

**关键字：** eNSP；设备巡检；Socket.IO；Telnet协议；MCP协议

## 第1章  课程设计概述

### 1.1 课程设计目的

随着网络技术的快速发展和企业对网络运维人才需求的不断增长，网络工程实践教学在高等教育中的地位日益重要。eNSP作为华为公司开发的企业网络模拟平台，广泛应用于网络工程课程的教学实验中。然而，传统的手工巡检方式存在以下问题：

（1）**效率低下**：网络工程师需要逐台登录设备，手动执行巡检命令，耗费大量时间和精力。

（2）**易出错**：人工操作难免出现遗漏或失误，影响巡检结果的准确性。

（3）**缺乏统一管理**：无法集中管理多台设备，巡检报告分散存储，难以追溯历史数据。

（4）**学习门槛高**：初学者需要熟悉各种命令行指令，学习成本较高。

本课程设计的目的是设计并实现一个智能化的eNSP设备管理与巡检平台，通过Web界面实现设备的自动发现、连接管理、命令执行和自动巡检功能，提高网络设备管理的效率和准确性，降低运维人员的工作负担，为网络工程实践教学提供有力支持。

### 1.2 课程设计内容

本课程设计的主要内容包括以下几个方面：

（1）**系统需求分析**：深入调研eNSP设备管理的实际需求，确定系统的功能范围和技术选型。

（2）**系统架构设计**：设计系统的整体架构，包括前端界面、后端服务、数据库存储和通信协议等。

（3）**核心功能开发**：

- 设备扫描模块：自动扫描指定端口范围，发现eNSP设备

- 设备连接模块：支持Telnet协议连接管理

- 命令执行模块：实时发送命令并获取响应

- 巡检报告模块：自动执行巡检命令并生成HTML格式报告

- 配置比对模块：比对设备配置差异

（4）**Web界面开发**：设计并实现响应式的Web用户界面，提供良好的用户体验。

（5）**MCP协议集成**：实现MCP服务器，使系统能够与AI助手集成，支持智能运维。

（6）**系统测试**：对各功能模块进行测试，验证系统的正确性和稳定性。

### 2.1 功能分析

经过详细的需求调研，本系统需要实现以下核心功能模块：

## 第3章  系统设计

### 3.1 总体结构设计

本系统采用B/S（Browser/Server）架构，分为前端展示层、业务逻辑层和数据访问层三个层次。

**前端展示层**负责用户界面的呈现和交互，包括设备列表展示、终端操作面板、巡检报告查看等功能模块。前端通过Socket.IO与后端建立WebSocket连接，实现实时通信。

**业务逻辑层**是系统的核心部分，使用Python Flask框架实现。主要包括：设备扫描服务、设备连接管理、命令执行引擎、巡检报告生成、数据持久化等功能模块。业务逻辑层通过Telnet协议与eNSP设备通信，通过HTTP接口向前端提供服务。

**数据访问层**负责系统数据的持久化存储。采用JSON文件存储拓扑数据和配置信息，HTML文件存储巡检报告，文件夹结构管理各种资源文件。

### 3.2 系统功能模块设计

**（1）设备扫描模块（DeviceScanner）**

功能：扫描指定端口范围的设备

主要方法：

- `scan_ports(start, end)`：扫描指定范围的端口，返回发现的设备列表

- `scan_devices(start, end)`：扫描设备并获取设备名称

- `get_connected_devices()`：获取当前已连接的设备列表

**（2）设备连接模块（TelnetConnection）**

功能：管理与eNSP设备的Telnet连接

主要方法：

- `connect(host, port)`：建立Telnet连接

- `send_cmd(command)`：发送命令并获取响应

- `close()`：关闭连接

**（3）设备管理模块（DeviceManager）**

功能：管理系统中的设备连接

主要方法：

- `connect_device(port)`：连接指定端口的设备

- `disconnect_device(path)`：断开设备连接

- `rename_device(path, name)`：重命名设备

- `send_command(path, command)`：向设备发送命令

**（4）巡检模块（Inspection）**

功能：自动执行设备巡检并生成报告

主要方法：

- `inspect_device(path)`：执行设备巡检，返回巡检结果

- `generate_inspection_report(device_info)`：生成HTML格式的巡检报告

- `save_inspection_report(report)`：保存巡检报告到文件

**（5）拓扑管理模块（TopologyManager）**

功能：管理网络拓扑数据

主要方法：

- `get_topology()`：获取当前拓扑数据

- `save_topology(data)`：保存拓扑数据

**（6）MCP服务模块（MCPServer）**

功能：提供MCP协议接口，供AI助手调用

主要方法：

- `list_tools()`：列出所有可用的工具函数

- `call_tool(name, arguments)`：调用指定的工具函数

### 4.3 设备扫描与连接模块实现

**（1）端口扫描实现**

使用Python的socket库实现端口扫描功能：

```python
def scan_ports(start=2000, end=2050):
    found = []
    for port in range(start, end + 1):
        try:
            s = socket.socket()
            s.settimeout(0.05)
            if s.connect_ex(('127.0.0.1', port)) == 0:
                found.append({'port': port, 'path': f'127.0.0.1:{port}'})
            s.close()
        except:
            pass
    return found
```

### 5.4 系统测试问题总结

在测试过程中发现并解决了以下问题：

**问题1：命令响应超时**

- 现象：某些命令执行后长时间无响应

- 原因：设备处理速度较慢，默认超时时间不足

- 解决方案：调整接收超时时间，配置命令超时为30秒

**问题2：中文显示乱码**

- 现象：设备输出中文信息显示为乱码

- 原因：Telnet通信使用GBK编码，系统默认使用UTF-8

- 解决方案：在解码时指定GBK编码：`response.decode('gbk', errors='ignore')`

**问题3：命令回显干扰**

- 现象：命令本身显示在输出中，影响阅读

- 原因：设备回显发送的命令，未正确过滤

- 解决方案：解析响应字符串，移除命令回显和提示符
