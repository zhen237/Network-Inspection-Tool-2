# eNSP MCP 项目实验报告

## 一、实验目的

1. 了解和掌握基于 Web 的网络设备管理工具的设计与实现
2. 学习使用 Flask + Socket.IO 构建实时 Web 应用
3. 掌握 Telnet 协议在网络设备管理中的应用
4. 学习设备巡检和配置比对功能的实现
5. 培养团队协作开发能力

## 二、实验环境

- **硬件环境**：PC 机一台
- **软件环境**：
  - Python 3.7+
  - Flask 2.0+
  - Flask-SocketIO 5.0+
  - eNSP 网络模拟平台
- **网络环境**：本地局域网

## 三、项目概述

**eNSP MCP** (Enterprise Network Simulation Platform Model Context Protocol) 是一个基于 Web 的 eNSP 设备管理工具，专为网络工程师和学习者设计。该工具通过 Web 界面提供了对 eNSP 模拟设备的集中管理、配置审计和故障排查功能。

### 主要功能特性

1. **设备管理**：设备扫描、连接、重命名和多设备管理
2. **终端操作**：命令执行、快捷命令按钮、命令历史记录和实时响应
3. **设备巡检**：自动巡检、生成 HTML 格式的巡检报告、历史报告查看
4. **配置比对**：通过 `compare configuration` 命令进行配置比对，结果追加到巡检报告
5. **拓扑管理**：支持上传和显示拓扑文件
6. **报告管理**：历史巡检报告的查看和下载

## 四、技术架构

### 前端
- HTML5 + CSS3 + JavaScript
- Socket.IO 实现实时通信
- 响应式布局设计
- 快捷命令按钮组件

### 后端
- Python Flask 框架
- Flask-SocketIO 处理实时通信
- Telnet 协议与设备通信
- 文件系统存储巡检报告和拓扑文件
- 新增 API 端点支持历史报告查看

### 通信流程
1. 前端通过 Socket.IO 与后端建立实时连接
2. 后端通过 Telnet 协议与 eNSP 设备通信
3. 命令执行结果通过 Socket.IO 实时返回给前端
4. 巡检报告和配置比对结果保存为 HTML 文件
5. 历史报告通过 REST API 查询和查看

## 五、实验步骤

### 1. 项目初始化
- 克隆项目代码
- 安装依赖包
- 启动服务器

### 2. 设备扫描与连接
- 设置扫描端口范围（默认 2000-2050）
- 点击「扫描」按钮扫描 eNSP 设备
- 连接到扫描到的设备

### 3. 设备巡检
- 点击设备列表中的「巡检」按钮
- 等待巡检完成，生成巡检报告
- 查看巡检报告内容

### 4. 配置比对
- 在终端中输入 `compare configuration` 命令或点击快捷按钮
- 执行命令并查看结果
- 打开巡检报告，查看「配置比对结果」部分

### 5. 历史报告查看
- 点击「历史报告」按钮
- 查看所有巡检报告列表
- 直接查看或下载历史报告

### 6. 拓扑管理
- 点击「上传拓扑」按钮
- 选择拓扑文件并上传
- 查看拓扑文件显示

## 九、技术难点与解决方案

### 1. 实时通信
- **难点**：实现前端与后端的实时通信，确保命令执行结果能够实时返回
- **解决方案**：使用 Flask-SocketIO 库，建立 WebSocket 连接，实现实时双向通信

### 2. Telnet 通信
- **难点**：处理 Telnet 连接的超时、错误和命令执行
- **解决方案**：实现健壮的 Telnet 通信模块，添加超时处理和错误捕获

### 3. 配置比对
- **难点**：实现配置比对功能，将结果追加到巡检报告
- **解决方案**：通过命令 `compare configuration` 执行比对，解析结果并追加到最新的巡检报告

### 4. 多设备管理
- **难点**：同时管理多个设备连接，处理并发操作
- **解决方案**：使用字典存储设备连接，实现标签页切换管理

### 5. 历史报告管理
- **难点**：实现历史巡检报告的查看和管理
- **解决方案**：新增 REST API 端点，支持报告列表查询和查看功能

### 6. 快捷命令实现
- **难点**：提供便捷的命令执行方式
- **解决方案**：在终端面板添加快捷按钮，一键执行常用命令
